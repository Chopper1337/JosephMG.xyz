Subnetting:

IP: 192.168.2.32/28



What is the valid range of hosts?

First determine the class of the IP:

Class A: 1-126
Class B: 128-191
Class C: 192-223
Class D: 224-239
Class E: 240-254


These ranges are in relation to the first octet of the IP, which in this case is "192".
That makes this a Class C address.

Now determine the class-full subnet of the IP:

Class A: 255.0.0.0
Class B: 255.255.0.0
Class C: 255.255.255.0
Class D: [RESERVED FOR MULTICASTING]
Class E: [EXPERIMENTAL; USED FOR RESEARCH]


As we have a Class C address, the class-full subnet is 255.255.255.0
but CIDR notation shows that this IP is classless.
In this case, we have more than 24 bits assigned for networks, 4 more bits have been added.


28 shows there are 28 1's in the subnet mask's binary
Here is the subnet mask's binary: 11111111111111111111111111110000
divided into octets: 11111111.11111111.1111111111.11110000


To get the last number in the subnet mask, consider each "1" in the subnet mask binary to be the following:
128 64 32 16 8 4 2 1

In the missing part of the subnet mask, we have the binary 11110000: 

1     1	   1	 1	  0	  0	  0	  0

128   64   32	 16	  8	  4	  2	  1


Now add the numbers which have a "1" above them,

128+64+32+16= 240

This results in the last number for the classless subnet mask:

255.255.255.240

Taking away 240 from 256 gives us the number of subnets

256-240= 16

Dividing the number of subnets by 256 gives us the range of hosts for each subnet:

256/16= 16

Then count up in 16's til you reach 256

16 - 32 - 48 - 64 - 80 - 96 - 112 - 128 - 144 - 160 - 176 - 192 - 208 - 224 - 240 - 256

These divisions are the ranges of hosts for all 16 subnets.

Going back to the IP, 192.168.2.32, it exists here:

16 - 32 - 48 - 64... ^-----^

32 and 48 are subnet identifiers which cannot be assigned as IP's, that means 192.168.2.33 is the first IP in this range.
192.168.2.47 is therefore the broadcast address for this subnet.

Therefore this is the valid range of IP's for this subnet:
192.168.2.33 to 192.168.2.46
